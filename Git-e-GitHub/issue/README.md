# Issues

## O que é?

Uma **Issue** é um **registro formal de uma necessidade relacionada ao projeto**.

Ela serve para **documentar, discutir, priorizar e acompanhar** algo que precisa ser feito.

Essa “necessidade” pode ser:
- uma **nova funcionalidade**
- um **bug**
- uma **melhoria**
- uma **dívida técnica**
- uma **pergunta técnica ou decisão de design**

Em projetos profissionais, a Issue é o **ponto inicial oficial do trabalho**.

Ela responde perguntas como:

- *O que precisa ser feito?*
- *Por que isso é necessário?*
- *Qual o impacto?*
- *Quando isso será considerado concluído?*

❌Sem Issue:

- o trabalho fica informal
- o histórico se perde
- decisões não ficam documentadas
